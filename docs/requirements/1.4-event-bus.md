# 1.4 事件总线（Event Bus）

## 核心职责
统一的事件发布/订阅接口，支持多种消息队列后端，实现组件间彻底解耦。

## 详细需求

| 需求项 | 规格 |
|--------|------|
| 抽象接口 | Publisher和Subscriber核心接口，与MQ实现解耦 |
| 后端适配器 | RabbitMQ、Redis Pub/Sub、内存三种 |
| 消息模型 | 任务分发(点对点)和事件广播(发布-订阅) |
| 公平分发 | RabbitMQ模式支持basicQos=1，按消费者处理速度分发 |
| 消息确认 | 手动ACK/NACK，NACK触发重试 |
| 消息持久化 | 支持持久化存储 |
| 事件结构 | {event_id, event_type, payload, timestamp, trace_id, source} |
| Schema验证 | 可选JSON Schema验证 |
| 死信处理 | 失败超限自动转入死信队列 |

## 对外接口

class EventBus(ABC):
    async def publish(self, event: Event, routing_key: str) -> None
    async def subscribe(self, event_type: str, handler: Callable, consumer_tag: str) -> None
    async def ack(self, event_id: str) -> None
    async def nack(self, event_id: str, requeue: bool = True) -> None
