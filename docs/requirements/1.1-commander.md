# 1.1 指挥官（Commander Engine）

## 核心职责
接收任务请求，解析工作流定义（DAG），拆解为子任务，发布到事件总线，监控执行进度。自身不执行业务逻辑。

## 详细需求

| 需求项 | 规格 |
|--------|------|
| 工作流定义 | YAML/JSON格式，包含Task节点、依赖边、条件分支、并行/串行标记 |
| 任务结构 | taskId、skillName、input、timeout、retryPolicy |
| 调度策略 | 串行、并行、条件分支(if/else)、扇出-扇入(fan-out/fan-in) |
| 任务分发 | 以事件发布到EventBus，不直接调用Agent |
| 进度跟踪 | 订阅TaskCompleted/TaskFailed事件，更新DAG状态 |
| 动态编排 | 运行时动态插入/移除/替换未执行的Task节点 |
| 超时处理 | 每个Task可配置超时，超时触发重试或失败回调 |
| 故障恢复 | 引擎无状态，重启后从EventBus重新订阅未完成事件 |
| 水平扩展 | 多实例通过Redis分布式锁保证同一工作流不被重复驱动 |

## 对外接口

class CommanderEngine:
    async def submit_workflow(self, workflow_def: WorkflowDef, input_data: dict) -> str
    async def get_status(self, run_id: str) -> WorkflowStatus
    async def cancel_workflow(self, run_id: str) -> bool
    async def insert_task(self, run_id: str, task: Task, after_task_id: str) -> bool

## 关键数据模型

class WorkflowDef(BaseModel):
    name: str
    version: str
    tasks: list[TaskDef]
    edges: list[Edge]

class TaskDef(BaseModel):
    task_id: str
    skill_name: str
    input_mapping: dict
    timeout: int = 300
    retry_policy: str = "default"
    depends_on: list[str] = []
