# 1.7 Harness 执行框架

## 核心职责
指挥官、仲裁器、状态机的协调容器。负责加载工作流定义，驱动整个执行生命周期，是框架的启动入口。

## 详细需求

| 需求项 | 规格 |
|--------|------|
| 组件装配 | 依赖注入组装Commander、Arbiter、StateMachine、EventBus、GatePipeline |
| 执行入口 | execute_workflow(name, input)统一入口 |
| 生命周期钩子 | on_start/on_complete/on_error/on_timeout回调 |
| 中间件链 | 可插拔中间件（日志、追踪、门禁、限流），洋葱模型 |
| 配置加载 | YAML/JSON配置，支持环境变量覆盖 |
| 插件发现 | 自动扫描并注册Skill和Agent |
| 优雅关闭 | SIGTERM信号处理，等待运行中任务完成 |

## 对外接口

class Harness:
    def __init__(self, config: HarnessConfig)
    async def start(self) -> None
    async def execute_workflow(self, name: str, input_data: dict) -> str
    async def shutdown(self) -> None
    def register_skill(self, skill: BaseSkill) -> None
    def use_middleware(self, middleware: Middleware) -> None
