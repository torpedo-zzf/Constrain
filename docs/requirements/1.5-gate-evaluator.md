# 1.5 分层门禁评估器（Gate Evaluator）

## 核心职责
可配置的多层质量门禁，支持硬性拦截、柔性放行、人工仲裁。

## 详细需求

| 需求项 | 规格 |
|--------|------|
| 三层架构 | L1硬性合规(自动拦截)、L2技术规范(柔性可特批)、L3创意评估(人工仲裁) |
| 规则引擎 | 规则为Callable或表达式，返回PASS/REJECT/NEED_REVIEW |
| 规则组合 | 同层内AND/OR组合 |
| L3人工流转 | NEED_REVIEW时暂停状态机，推送人工审核，等待回调 |
| 置信度阈值 | L2可配置，高于阈值的REJECT转NEED_REVIEW |
| 反馈闭环 | 人工裁定结果记录，支持定期回训练阈值模型 |
| 审计日志 | 输入、输出、决策时间、决策者全量记录 |

## 对外接口

class GatePipeline:
    def add_layer(self, layer: GateLayer) -> None
    async def evaluate(self, content: dict, trace_context: TraceContext) -> GateResult

class GateVerdict(Enum):
    PASS = "pass"
    REJECT = "reject"
    NEED_REVIEW = "need_review"
