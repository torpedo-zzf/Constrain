# 1.3 状态机（State Machine）

## 核心职责
管理所有执行单元的生命周期状态，严格约束状态转移路径，持久化状态变更事件。

## 详细需求

| 需求项 | 规格 |
|--------|------|
| 状态定义 | 用户自定义，预置PENDING→RUNNING→SUCCEEDED/FAILED/PAUSED/CANCELLED |
| 转移约束 | 严格校验合法性，非法转移抛异常 |
| 事件驱动 | 状态变更自动发布StateChangeEvent到EventBus |
| 持久化 | 事件日志写入存储，支持Event Sourcing |
| 状态恢复 | 从事件日志重建当前状态 |
| 暂停/恢复 | PAUSE/RESUME操作，暂停保留完整上下文 |
| 交互等待 | 预置WAITING_USER_INPUT，等待外部信号恢复 |
| 超时处理 | 状态级别超时配置，超时自动转TIMEOUT |

## 对外接口

class StateMachine:
    async def create_instance(self, entity_type: str, entity_id: str, initial_state: str, context: dict) -> None
    async def transition(self, entity_id: str, to_state: str, event_data: dict) -> StateChangeEvent
    async def get_current_state(self, entity_id: str) -> State
    async def get_state_history(self, entity_id: str) -> list[StateChangeEvent]
    async def pause(self, entity_id: str, reason: str) -> None
    async def resume(self, entity_id: str) -> None
